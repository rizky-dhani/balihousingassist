## ADDED Requirements

### Requirement: Footer Component Existence
The application MUST include a footer component that is consistently displayed across all pages.

#### Scenario: Basic Footer Structure
The application MUST include a footer component that is consistently displayed across all pages.

#### Scenario: Basic Footer Structure
The application MUST include a footer component that is consistently displayed across all pages.

### Requirement: Section Headers
The footer MUST prominently display three main sections with the following headers: "Navigation", "Properties", and "Helpful Links".

#### Scenario: Section Headers
The footer MUST prominently display three main sections with the following headers: "Navigation", "Properties", and "Helpful Links".

### Requirement: Dynamic Navigation Links
The "Navigation" section of the footer MUST display a list of links sourced from a shared configuration, mirroring the navigation items present in the application's header.

#### Scenario: Dynamic Navigation Links
The "Navigation" section of the footer MUST display a list of links sourced from a shared configuration, mirroring the navigation items present in the application's header.

### Requirement: Dynamic Property Categories
The "Properties" section of the footer MUST dynamically list available property categories. Each category name MUST be a clickable link.

#### Scenario: Dynamic Property Categories
The "Properties" section of the footer MUST dynamically list available property categories. Each category name MUST be a clickable link.

### Requirement: "Contact Us" Link
The "Helpful Links" section of the footer MUST contain a "Contact Us" link. This link MUST direct users to a WhatsApp chat using the `whatsapp_number` retrieved from the `SiteSetting` model.

#### Scenario: "Contact Us" Link
The "Helpful Links" section of the footer MUST contain a "Contact Us" link. This link MUST direct users to a WhatsApp chat using the `whatsapp_number` retrieved from the `SiteSetting` model.

### Requirement: "FAQs" Link
The "Helpful Links" section of the footer MUST contain a "FAQs" link. This link MUST direct users to an appropriate FAQs page (initially, this can be a placeholder).

#### Scenario: "FAQs" Link
The "Helpful Links" section of the footer MUST contain a "FAQs" link. This link MUST direct users to an appropriate FAQs page (initially, this can be a placeholder).
